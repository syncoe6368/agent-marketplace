## 3. Agents Table

| Policy | Operation | Condition | Verdict |
|--------|-----------|-----------|---------|
| Active agents are viewable by everyone | SELECT | `status = 'active'` | ✅ Correct — hides pending/suspended |
| Creators can view own agents | SELECT | `auth.uid() = creator_id` | ✅ Correct — creators see their own drafts |
| Authenticated users can create agents | INSERT | `auth.uid() = creator_id` | ✅ Correct — prevents impersonation |
| Creators can update own agents | UPDATE | `auth.uid() = creator_id` | ✅ Correct |
| Creators can delete own agents | DELETE | `auth.uid() = creator_id` | ✅ Correct |

**Gaps Found:**

- ⚠️ **MEDIUM: No admin override policies.** If an admin needs to moderate (suspend/feature/verify), they can't do it through the API without direct DB access. 
  - **Recommendation:** Add admin role check policies or use Supabase service_role key in admin API routes.
  
- ⚠️ **LOW: Creator can change status to 'active' directly.** A creator could bypass moderation by setting `status = 'active'` on insert/update.
  - **Recommendation:** Add a BEFORE UPDATE trigger that prevents non-admin users from changing status from 'pending' to 'active'.

- ⚠️ **LOW: No rate limiting on agent creation.** A malicious user could create unlimited agent listings.
  - **Mitigation:** API-level rate limiting exists (20 writes/min), but no DB-level constraint.

## 7. General Security Observations

### ✅ Good Practices Found:
- RLS enabled on ALL tables
- Foreign key constraints prevent orphan records
- Unique constraints prevent duplicate reviews and version entries
- `updated_at` auto-triggers for audit trails
- Service role key NOT used in client-facing API routes
- Slug generation prevents name collisions

### ⚠️ Recommendations:

1. **HIGH: Add admin role to profiles table** and create admin-specific RLS policies:
   ```sql
   ALTER TABLE public.profiles ADD COLUMN role text DEFAULT 'user' CHECK (role IN ('user', 'admin'));
   
   CREATE POLICY "Admins can update any agent status"
     ON public.agents FOR UPDATE
     USING (EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin'));
   ```

2. **MEDIUM: Add BEFORE INSERT/UPDATE trigger to prevent self-activation:**
   ```sql
   CREATE OR REPLACE FUNCTION prevent_self_activation()
   RETURNS TRIGGER AS $$
   BEGIN
     IF NEW.status = 'active' AND OLD.status = 'pending' THEN
       -- Only allow if user is admin
       IF NOT EXISTS (SELECT 1 FROM profiles WHERE id = auth.uid() AND role = 'admin') THEN
         NEW.status := 'pending';
       END IF;
     END IF;
     RETURN NEW;
   END;
   $$ LANGUAGE plpgsql SECURITY DEFINER;
   ```

3. **LOW: Add index on skill_update_subscriptions for slug lookups:**
   ```sql
   CREATE INDEX idx_skill_subscriptions_slug ON public.skill_update_subscriptions(slug) WHERE is_active = true;
   ```
